Profiler Configuration
Code Decomposition

Summary of what the code does, how many lines of code, number of classes, average number of lines of code per class, number of programs more than 2000 lines of code, number of functions and methods, redundent code, legacy libraries, dependencies, unit test code coverage, number of transaction screens, number of dashboards and operational reports.  

Extract Business Logic

Over the years, the application may have business logic that is hard to read and understand but needs to be preserved.  

Identify Business Processes

Evaluate if application has organized workflow, understand Level Zero Business Processes and other business automations and create flows.  

Identify Security and Performance Issues

Scan for potential security and performance issues within code, including SQL injections, cross site scripting, hard coded passwords, long running queries, unnecessary loops, etc.  

Identify Integration Point

Identify 3rd party integration point and its complexity, as far as, number of calls to the same 3rd party, complexity of json/xml payloads being exchanged, sync vs call back endpoints, method of authentication, etc.  

Database Analysis

Number of database tables, views, stored procedures and their complexity, number of lines of code, number of stored procedures more than 500 lines of code, proper use of indices, referential integrity enforced by primary and foreign keys, etc.  

Code Compatibility

Is the code fit to auto-modernize to desired target source language, is current code fit for CI/CD, is current code fit for containerization, is the code fit for Microservice architecture, etc.
