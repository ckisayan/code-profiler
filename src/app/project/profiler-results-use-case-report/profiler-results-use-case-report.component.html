Functional System Use Cases

Use Case 1: Display Shipment Subfile

Description Display a subfile of shipment records on the screen.
Primary Actor User
Preconditions
  • The ShipPf file must contain shipment records.
  • The Shipdspf1 display file must be available.
Main Flow
  1. User initiates the display of shipment records.
  2. System clears the subfile.
  3. System loads shipment records from ShipPf into the subfile.
  4. System displays the subfile control record.
  5. User can navigate through the subfile pages.
Postconditions Shipment records are displayed on the screen.
Includes Clear Subfile, Load Subfile

Use Case 2: Add New Shipment Record

Description Add a new shipment record to the ShipPf file.
Primary Actor User
Preconditions
  • The ShipPf file must be available.
  • The Shipdspf1 display file must be available.
Main Flow
  1. User selects the option to add a new shipment.
  2. System displays the add record window.
  3. User enters shipment details (ID, Description, Date).
  4. System validates the entered details.
  5. If validation passes, system writes the new record to ShipPf.
  6. System displays a success message.
Postconditions A new shipment record is added to the ShipPf file.
Includes Validate Shipment Details

Use Case 3: Display Customer Order Entry & Change Menu

Description Display the menu for customer order entry and change.
Primary Actor User
Preconditions
  • The custord display file must be available.
Main Flow
  1. User initiates the display of the customer order menu.
  2. System displays the customer order entry & change menu.
Postconditions The customer order entry & change menu is displayed.
